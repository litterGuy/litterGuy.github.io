# git设置双源修改合并代码

##  克隆 GitHub 仓库 + 添加 GitLab 远程
```
# 克隆 GitHub 项目，并将其 origin 指向 GitHub 源
git clone https://github.com/langgenius/dify.git
cd dify

git remote add gitlab https://gitlab.com/你的用户名/dify.git

# 验证 remotes 设置
git remote -v
# 应显示 origin 和 gitlab 两个远程
```

## 查看tag
```
git tag
```

## 根据tag创建分支
```
git checkout -b <新分支名> <标签名>

如：
git checkout -b master 1.7.2
```

## 提交代码到gitlab
```
git push gitlab master

git push gitlab main

```

## 提交变动
```
git commit -m "docs: 修改readme"
git push gitlab master
```

## 同步最新github代码
```
# 首先确保在你希望合并这些更新的分支上
git checkout main

# 拉取 GitHub 上游改动
git fetch origin
```

## 合并分支
```
# 切换到目标分支
git checkout master

# 将github的main分支代码合并到gitlab的masterr分支
git merge main

# 提交并推送
git push gitlab master
git push gitlab main
```
